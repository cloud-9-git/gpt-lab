Dataset / InputEmbedding 구현 학습

src/dataset.py와 src/embeddings.py를 구현하기 위한 문서입니다. GPT 사전 학습용 input/target 쌍을 만들고, token ID를 Transformer 입력 벡터로 바꾸는 흐름을 다룹니다.

1. 목표 2. 다음 토큰 예측 3. Dataset 길이 4. __getitem__ 5. DataLoader 6. InputEmbedding 7. 테스트 기준 8. 자주 나는 실수

1. 목표

BPE tokenizer가 문장을 token ID 리스트로 바꿔주면, 그 다음에는 이 긴 리스트를 GPT가 학습할 수 있는 작은 조각으로 잘라야 합니다. 이 역할을 GPTDataset이 합니다.

그리고 Dataset이 만든 token ID tensor는 아직 그냥 정수 번호입니다. Transformer는 정수 번호를 바로 계산하지 않고, 각 token ID를 벡터로 바꿔서 사용합니다. 이 역할을 InputEmbedding이 합니다.

파일 구현 대상 역할
src/dataset.py GPTDataset, create_dataloader token ID 리스트를 input/target 학습 샘플과 batch로 만듦
src/embeddings.py InputEmbedding token ID를 token embedding + position embedding 벡터로 바꿈

2. GPT의 다음 토큰 예측

GPT는 현재 token들을 보고 다음 token을 맞히도록 학습합니다. 그래서 input과 target은 거의 같지만, target이 input보다 한 칸 뒤에 있습니다. 

token_ids = [10, 11, 12, 13]
context_length = 3

input  = [10, 11, 12]
target = [11, 12, 13]

모델 입장에서는 이런 문제가 됩니다.

현재 위치 input 맞혀야 하는 target
10 11
11 12
12 13
이 단계의 핵심 검사는 target[:-1] == input[1:]입니다. target이 input보다 한 칸 뒤라는 뜻입니다.

3. GPTDataset 길이 계산

Dataset의 길이는 만들 수 있는 학습 샘플 개수입니다. input 길이가 context_length이고 target도 같은 길이입니다. 그런데 target은 input보다 한 칸 뒤에서 끝나므로, 샘플 하나를 만들려면 원본 token이 context_length + 1개 필요합니다. 

token_ids = [0, 1, 2, 3, 4]
context_length = 4

input  = [0, 1, 2, 3]
target = [1, 2, 3, 4]

필요한 원본 token 개수 = 5 = context_length + 1

stride

stride는 다음 샘플로 넘어갈 때 시작 위치를 몇 칸 옮길지 정합니다. stride=1이면 샘플이 많이 겹치고, stride=context_length이면 거의 겹치지 않습니다. 

token_ids = list(range(12))
context_length = 4
stride = 4

sample 0:
input  = [0, 1, 2, 3]
target = [1, 2, 3, 4]

sample 1:
input  = [4, 5, 6, 7]
target = [5, 6, 7, 8]

테스트에서 기대하는 길이 공식은 아래와 같습니다. 

expected = (len(token_ids) - context_length - 1) // stride + 1

token 수가 너무 적으면 음수가 나올 수 있으므로, 실제 구현에서는 max(0, ...)로 감싸면 안전합니다. 

available = len(token_ids) - context_length - 1
self._length = max(0, available // self.stride + 1)

4. __getitem__ 구현

__getitem__(idx)는 idx번째 input/target 쌍을 반환합니다. 시작 위치는 idx * stride입니다. 

start = idx * self.stride
end = start + self.context_length

input_ids = self.token_ids[start:end]
target_ids = self.token_ids[start + 1:end + 1]

반환할 때는 Python list가 아니라 PyTorch tensor로 바꿉니다. token ID는 embedding에 들어가는 정수 index이므로 dtype은 torch.long이어야 합니다. 

return (
    torch.tensor(input_ids, dtype=torch.long),
    torch.tensor(target_ids, dtype=torch.long),
)
torch.long은 정수 ID용 dtype입니다. nn.Embedding은 float tensor가 아니라 정수 index tensor를 입력으로 받습니다.

5. create_dataloader

Dataset은 샘플 하나를 꺼내는 객체입니다. DataLoader는 Dataset에서 여러 샘플을 모아 batch로 만들어 줍니다. 

dataset = GPTDataset(token_ids, context_length, stride=stride)

loader = DataLoader(
    dataset,
    batch_size=batch_size,
    shuffle=shuffle,
    drop_last=drop_last,
    num_workers=num_workers,
)

예를 들어 샘플 하나의 input shape가 (16,)이고 batch_size=4이면, DataLoader가 반환하는 input batch shape는 (4, 16)입니다.

6. InputEmbedding

Dataset/DataLoader가 만든 batch는 shape가 (batch_size, seq_len)인 token ID tensor입니다. 하지만 Transformer는 token ID 숫자 자체가 아니라 벡터를 입력으로 받습니다.

token embedding

token ID를 emb_dim 차원의 벡터로 바꿉니다. 

self.token_embedding = nn.Embedding(vocab_size, emb_dim)

position embedding

GPT는 token의 순서도 알아야 합니다. 같은 token이라도 0번째 위치인지, 5번째 위치인지에 따라 문맥이 다르기 때문입니다. 

self.position_embedding = nn.Embedding(context_length, emb_dim)

forward 흐름

batch_size, seq_len = x.shape

tok_emb = self.token_embedding(x)
# shape: (batch_size, seq_len, emb_dim)

pos_ids = torch.arange(seq_len, device=x.device)
pos_emb = self.position_embedding(pos_ids)
# shape: (seq_len, emb_dim)

out = tok_emb + pos_emb
out = self.dropout(out)
# shape: (batch_size, seq_len, emb_dim)

tok_emb는 batch 차원이 있고 pos_emb는 batch 차원이 없습니다. 그래도 PyTorch가 broadcasting으로 모든 batch에 같은 position embedding을 더해줍니다.

테스트가 확인하는 최종 출력 shape는 (batch_size, seq_len, emb_dim)입니다.

8. 자주 나는 실수

target을 input과 똑같이 만드는 실수

# 틀림
target_ids = self.token_ids[start:end]

# 맞음
target_ids = self.token_ids[start + 1:end + 1]

길이 계산에서 -1을 빼먹는 실수

target까지 만들려면 input보다 token 하나가 더 필요합니다. 

(len(token_ids) - context_length - 1) // stride + 1

Tensor dtype을 float로 만드는 실수

torch.tensor(input_ids, dtype=torch.long)

position id를 CPU에 만드는 실수

나중에 GPU 학습에서 device mismatch가 날 수 있으므로 x.device를 사용합니다. 

pos_ids = torch.arange(seq_len, device=x.device)
